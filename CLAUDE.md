# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

A Claude Code plugin that gives Claude access to a full design suite: 58 brand design systems + 5 curated external skills (UI/UX, web guidelines, motion, a11y, full design process). Skills are auto-selected based on project type via a SessionStart hook.

**Design data source:** [VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md) — cloned locally by `sync.sh`, not committed to this repo.

## Setup / Sync

```bash
# Clone this plugin into Claude Code's plugins directory
git clone https://github.com/zeta92/design-library-plugin.git ~/.claude/plugins/local/design-library

# Run once after install: clones design data and installs a daily 6am cron job
bash scripts/sync.sh
```

After sync, design files live at:
```
designs/awesome-design-md/design-md/<brand>/DESIGN.md
```

## Plugin architecture

| File | Role |
|------|------|
| `.claude-plugin/plugin.json` | Plugin metadata (name, version, description) |
| `commands/design.md` | `/design` slash command — defines all argument modes and behavior |
| `hooks/hooks.json` | Registers `UserPromptSubmit` hook pointing to `inject-context.sh` |
| `hooks/scripts/inject-context.sh` | On every prompt: checks for design intent keywords, matches brand names, injects DESIGN.md content as `systemMessage` |
| `scripts/sync.sh` | Clones/updates `designs/awesome-design-md`, regenerates `catalog.md`, installs cron |
| `hooks/scripts/detect-project.sh` | SessionStart hook — scans project, activates relevant skills |
| `skills/<name>/SKILL.md` | Thin wrappers pointing to skill content in `designs/` |
| `commands/ui-ux.md`, `commands/guidelines.md`, `commands/motion.md`, `commands/a11y.md`, `commands/design-process.md` | 5 new skill commands |
| `skills/design-library/SKILL.md` | Skill that defines how to read, apply, and mix DESIGN.md files |
| `skills/design-library/references/catalog.md` | Auto-generated brand list — **do not edit manually**, regenerated by `sync.sh` |

## How the hook works

`inject-context.sh` runs on every user prompt:
1. Exits early if no design intent keywords are found (fast path — avoids overhead on non-design prompts)
2. Scans available brand directories against the prompt text (case-insensitive)
3. If brands matched: reads each `DESIGN.md` and outputs a `systemMessage` JSON payload with full design context
4. If no brand matched but design intent detected: outputs a hint that the library is available

Design intent keywords (EN + ES): `diseña`, `diseñame`, `estilo`, `interfaz`, `componente`, `landing`, `design`, `style`, `interface`, `component`, `ui`, `frontend`, `como.*google`, `como.*stripe`, `like.*figma`.

## `/design` command modes

| Argument | Behavior |
|----------|----------|
| `<brand>` | Load one brand's DESIGN.md |
| `A + B` | Mix two brands (default section assignment) |
| `A:section + B:section` | Granular mix by section |
| `?` | Auto-suggest based on project context |
| `list` | Display catalog grouped by category |
| `sync` | Run `sync.sh` |
| `/ui-ux` | Generate complete design system (ui-ux-pro-max, 61k ⭐) |
| `/guidelines` | Audit UI code: 100+ rules (vercel web-design-guidelines, 24k ⭐) |
| `/motion` | Motion audit via 3 designer perspectives (256 ⭐) |
| `/a11y` | WCAG 2.2 AA audit via 11 specialist agents (214 ⭐) |
| `/design-process` | Full design lifecycle: 63 skills (Owl-Listener, 505 ⭐) |
| _(none)_ | Show usage help |

Valid sections for granular mix: `colors`, `typography`, `components`, `layout`, `elevation`.

Default section assignment when mixing A + B: Colors/Components/Elevation → Brand A; Typography/Layout → Brand B.

## DESIGN.md structure

Each brand's DESIGN.md has 9 sections: Visual Theme & Atmosphere, Color Palette & Roles, Typography Rules, Component Stylings, Layout Principles, Depth & Elevation, Design Do's and Don'ts (or Interaction/Motion/Dark Mode — varies by brand), Responsive Behavior, Agent Prompt Guide.

> Section titles vary across brands — always read actual headers rather than assuming fixed names.

## Notes

- `catalog.md` is regenerated every time `sync.sh` runs — it reflects the actual directory listing of the local clone.
- The plugin is installed globally at `~/.claude/plugins/local/design-library/`. Paths in `commands/design.md` and `skills/design-library/SKILL.md` reference this install location.
- The `designs/` directory is in `.gitignore` (created at runtime by `sync.sh`).
